\section{Introduction}
\label{sec:intro}

LLM agents now trigger irreversible effects: sending messages, invoking external APIs, and handling payment-like workflows.
In this setting, prompt injection and tool-chain compromise are no longer only ``reasoning errors''; they become execution-time security failures.
Recent evidence shows that simple prompt injection can make tool-calling agents leak personal data observed during task execution \citep{alizadeh2025simple}.
At the protocol layer, AP2-style agent payments further raise the stakes because manipulated agent actions can directly affect financial outcomes \citep{ap2spec2025,whisperswealth2026}.

A natural reaction is to add centralized policy control (AI gateway, DLP, LLM firewall).
However, this introduces a second failure mode: the same control plane that enforces policy may observe and log sensitive query values.
We call this the \emph{control-plane privacy paradox}.

\paragraph{Thesis.}
In agentic tool-use systems, robust security under compromised runtimes requires a \emph{cryptographically non-bypassable effect boundary}, and deploying this boundary at scale requires \emph{single-auditor privacy (SAP)} so outsourced policy checks do not disclose sensitive query values to any single policy server.

\paragraph{Contributions.}
\begin{itemize}
  \item \textbf{C1: Non-bypassable effects (core).}
  We define and implement a PREVIEW$\rightarrow$COMMIT protocol in which no external side effect commits unless the executor verifies dual commit-proof shares bound to a canonical request hash (including delegation context), freshness window, and replay state.

  \item \textbf{C2: Control-plane privacy paradox and SAP (core).}
  We formalize outsourced policy privacy with an explicit leakage contract $L_{\mathsf{policy}}=(L_{\mathsf{PIR}},L_{\mathsf{MPC}},L_{\mathsf{CONFIRM}},L_{\mathsf{TIME}})$ and state a simulation-based single-auditor theorem: each single policy-server transcript is simulatable from $L_{\mathsf{policy}}$ under non-collusion and fixed-shape routing.

  \item \textbf{C3: Confidentiality and multi-principal correctness under compromised runtimes (core).}
  We compose handle-first plaintext confinement, sanitize-at-commit integrity, capsule-conditional skill confinement, and delegation-aware hash-context binding into one system-level theorem with explicit assumption partitioning.

  \item \textbf{C4: Claim-aligned evidence and machine-checkable claim gates.}
  On official C1--C5 coverage, SecureClaw reaches $0.00\%$ observed attack leakage and $100.00\%$ attack blocking with $80.00\%$ benign allow.
  In SAP distinguishability sweeps, shaping collapses single-server intent signal from separable (PIR MI $0.414$ bits) to near-zero.
  We additionally expose versioned, machine-checkable accept-predicate and capsule-contract verifiers that enforce claim-gating and reduced-claim downgrade semantics.
\end{itemize}

\section{Background and Motivation}
\label{sec:background}

\subsection{Agent leakage surfaces}

Agent pipelines expose multiple leakage channels: final responses, inter-agent messages, tool arguments/results, and memory interfaces.
Prompt injection and indirect context contamination exploit these channels across a wide class of benchmarks and real tool-use settings \citep{greshake2023indirect,injecagent2024,agentdojo2024,agentleak2026}.

\subsection{The control-plane privacy paradox}

Operationally, enterprises often centralize policy checks for safety, auditing, and compliance.
Yet centralized control planes may inspect and retain prompt and tool payloads \citep{azureapimllmlogs2026,cloudflarefirewallai2026,cloudflareaidlp2026,dynatracemasklogs2026}.
Thus, a defense layer can become a high-value privacy sink.
SecureClaw addresses this paradox directly: policy outsourcing remains centralized in function, but single-server transcript visibility is leakage-contracted and simulatable.

\section{Threat Model and Security Goals}
\label{sec:threat}

\subsection{Parties and trust assumptions}

We model five roles:
(1) adversarial runtime $\mathcal{A}$ (controls prompts, tool invocations, and skill code),
(2) trusted gateway $\mathcal{G}$,
(3) policy servers $\mathcal{P}_0,\mathcal{P}_1$,
(4) trusted executor $\mathcal{X}$ (sole side-effect sink), and
(5) user confirmation source $\mathcal{U}$.

For SAP, we assume non-collusion between $\mathcal{P}_0$ and $\mathcal{P}_1$ (single-auditor threat).
If both collude, single-auditor query privacy is impossible in this model.

\subsection{Goals and non-goals}

\textbf{Goals.}
\begin{itemize}
  \item \textbf{Effect integrity (NBE):} no commit without valid dual, request-bound proof shares.
  \item \textbf{Control-plane privacy (SAP, optional mode):} single policy-server transcript leaks only $L_{\mathsf{policy}}$.
  \item \textbf{Confidentiality and context correctness:} plaintext is not exposed to $\mathcal{A}$ by default; commits remain bound to caller/session/principal/delegation context.
\end{itemize}

\textbf{Non-goals.}
\begin{itemize}
  \item No claim if gateway or executor is compromised.
  \item No claim against kernel/hardware side channels.
  \item No SAP claim under full policy-server collusion.
\end{itemize}

\section{Design Overview}
\label{sec:overview}

SecureClaw separates planning from effects.
Runtime logic may propose actions through a single high-level interface, but external side effects are committed only by the executor.
The gateway mediates policy preview, handleization/declassification, delegation checks, and (in SAP mode) fixed-shape policy outsourcing.

At a high level:
\begin{enumerate}
  \item Runtime calls \texttt{act(intent\_id, inputs, constraints, caller)}.
  \item Gateway canonicalizes and computes request binding hash.
  \item Policy path returns dual commit-proof shares plus semantic bits.
  \item Executor verifies accept predicate and either fail-closes or commits effect.
\end{enumerate}

\section{Non-Bypassable Effects Protocol}
\label{sec:nbe}

\subsection{Canonical request binding}

For effectful tuple
$\rho=(\mathsf{intent\_id},\mathsf{caller},\mathsf{session},\mathsf{inputs},\mathsf{hctx})$
with
$\mathsf{hctx}=(\mathsf{external\_principal},\mathsf{delegation\_jti})$,
we define
\[
\mathsf{ReqHash}(\rho)=\mathrm{SHA256}(\mathsf{CanonJSON}(\rho)).
\]
Both gateway and executor use the same canonical serializer.

\subsection{PREVIEW$\rightarrow$COMMIT and accept predicate}

PREVIEW obtains dual proof shares from independent policy servers.
COMMIT succeeds iff executor verifies all of:
\begin{enumerate}
  \item two valid, distinct-server MAC-authenticated commit proofs,
  \item freshness within TTL,
  \item agreement on $(\mathsf{action\_id},\mathsf{program\_id},\mathsf{request\_sha256})$,
  \item recomputed request-hash equality (including $\mathsf{hctx}$),
  \item replay non-membership,
  \item semantic constraints (allow/confirm/sanitize).
\end{enumerate}
Any failure is fail-closed.

\subsection{Security statement}

Theorem \ref{thm:nbe} (Appendix \ref{app:proofs}) reduces commit bypass to MAC forgery, hash collision/serialization inconsistency, or explicit operational assumption failures (freshness/replay-state assumptions).

\section{Control-Plane Privacy Paradox and SAP}
\label{sec:sap}

\subsection{Leakage contract}

We model single-server observable leakage as
\[
L_{\mathsf{policy}}=(L_{\mathsf{PIR}},L_{\mathsf{MPC}},L_{\mathsf{CONFIRM}},L_{\mathsf{TIME}}).
\]
Allowed terms include fixed endpoint class, public program id/circuit shape, padded batch occupancy, and coarse timing buckets.
Forbidden terms include true PIR indices and plaintext predicate inputs.
Intent hiding is claimed only when public observables (program id, endpoint class, batch geometry) are constant across covered intents.

\subsection{Single-auditor privacy game}

Adversary chooses workloads $(W_0,W_1)$ with identical leakage:
$L_{\mathsf{policy}}(W_0)=L_{\mathsf{policy}}(W_1)$.
Challenger samples $b\leftarrow\{0,1\}$, executes $W_b$, and reveals one policy-server view $\mathsf{View}(\mathcal{P}_\sigma)$.
Adversary outputs $b'$.
SAP requires negligible distinguishing bias over $1/2$.

\subsection{Theorem and interpretation}

Theorem \ref{thm:sap} states that under single-server PIR privacy, HBC 2PC privacy, non-collusion, and fixed-shape scheduling, each single-server transcript is simulatable from $L_{\mathsf{policy}}$.
Interpretation: one policy server cannot recover sensitive query values beyond explicit allowed leakage.

SAP does \emph{not} protect against:
(i) full policy-server collusion,
(ii) intentionally unshaped deployment where intent-correlated metadata is left observable,
and (iii) leakage channels outside the policy transcript model.

\section{Formal Security}
\label{sec:formal}

We state one system-level proposition and prove it via component theorems (Appendix \ref{app:proofs}).

\paragraph{System proposition.}
For deployed SecureClaw interfaces, no external side effect commits without dual request-bound commit evidence; plaintext release and cross-principal replay remain constrained to explicit interfaces and assumptions; and in SAP mode, single-server transcript leakage is bounded by $L_{\mathsf{policy}}$.

\paragraph{Component theorems.}
\begin{itemize}
  \item \textbf{NBE} (Theorem \ref{thm:nbe}): commit integrity under dual-proof, hash binding, freshness, replay assumptions.
  \item \textbf{SM/PEI/SCS/DAS} (Theorems \ref{thm:sm}, \ref{thm:pei}, \ref{thm:scs}, \ref{thm:das}): confidentiality, sanitize-at-commit integrity, capsule-conditional confinement, and delegation-aware binding.
  \item \textbf{SAP} (Theorem \ref{thm:sap}, optional mode): single-auditor transcript privacy.
\end{itemize}

\paragraph{Composition bound.}
Let $\mathsf{Bad}_{\mathsf{NBE}},\mathsf{Bad}_{\mathsf{SM}},\mathsf{Bad}_{\mathsf{PEI}},\mathsf{Bad}_{\mathsf{SCS}},\mathsf{Bad}_{\mathsf{DAS}},\mathsf{Bad}_{\mathsf{SAP}}$ denote property-specific bad events.
Then (Theorem \ref{thm:composition}):
\[
\Pr[\mathsf{Exp}^{\mathsf{SC}}=1]
\le
\sum_{E \in \{\mathsf{NBE},\mathsf{SM},\mathsf{PEI},\mathsf{SCS},\mathsf{DAS},\mathsf{SAP}\}}
\Pr[\mathsf{Bad}_E].
\]
No mutual exclusivity assumption is needed; overlap only affects tightness.

\section{Implementation}
\label{sec:impl}

SecureClaw is implemented as a gateway-executor architecture with machine-checkable protocol specs.
Core implementation anchors include:
\begin{itemize}
  \item canonical binding and hash computation,
  \item machine-checkable protocol contracts (versioned schema + semantic accept-predicate spec),
  \item dual-proof verification and replay guard in executor,
  \item fixed-shape PIR/MPC routing for SAP mode,
  \item capsule contract verifier with mandatory-assertion claim gating and downgrade semantics,
  \item formal harnesses for security games and model checks.
\end{itemize}

Artifact-level validators include:
\texttt{scripts/security\_game\_nbe\_check.py},
\texttt{formal/secureclaw\_model\_check.py},
and
\texttt{scripts/validate\_specs.py}.

\section{Evaluation}
\label{sec:eval}

\subsection{RQs and threat-model split}

We evaluate four measured questions and three extension protocols.

\textbf{Threat Model A (honest runtime).}
Native guardrail baselines (Codex/OpenClaw and mediated DRIFT/IPIGuard/AgentArmor) are interpreted as runtime-behavior defenses where direct executor bypass is out of scope.

\textbf{Threat Model B (compromised runtime).}
Runtime can attempt direct effect bypass, replay/context swapping, and exfil paths outside intended tool mediation.
SecureClaw's NBE/SCS claims are evaluated here.

\paragraph{Query-value privacy sanity check.}
To separate \emph{intent-shape} leakage from \emph{query-value} leakage, we run a 16-way recipient-index task on single-server transcript features.
Observed single-server recipient prediction remains at or below chance (chance $=6.25\%$; unshaped $2.89\%$, shaped-pad4-cover $4.50\%$) with near-zero MI.
This is consistent with PIR single-server index privacy: shaping is primarily needed to suppress intent-correlated metadata rather than to hide PIR query indices.

\subsection{RQ3: delegated multi-agent safety}
\label{sec:rq3}

In a deterministic delegated campaign (7 cases), all outcomes match expected policy semantics (7/7 pass), including missing-delegation denial, revocation denial, dual-principal denial, and mediated memory access.
Gateway check latency remains low (median $0.38$ ms, p95 $5.56$ ms).

\subsection{RQ4: systems cost and fairness}
\label{sec:rq4}

We separate enforcement overhead from model-inference latency.
Shaping overhead is decomposed into (i) padding/cover compute, (ii) mixer queueing, and (iii) transport framing.
Executor verification remains constant-sized (dual MAC verify + replay lookup).

\subsection{E1 (completed): compromised-runtime bypass suite}
\label{sec:e1}

We evaluate direct bypass attempts: missing proof share, one-server proof only, request/session/caller/hctx mismatches, replay, and downgraded baselines.
The current artifact report (\texttt{compromised\_bypass\_report.json}) records 16/16 expected outcomes, including fail-closed denial for bypass attempts and explicit bypass observability in intentionally insecure baselines.

\subsection{E2 (completed): single-auditor compromise simulation}
\label{sec:e2}

The SAP sweep operationalizes a single-policy-server adversary using transcript-only features, MI, and classifier attacks.
This directly instantiates the SAP game with empirical distinguishers under controlled shaping ablations.

\subsection{E3--E6 (protocolized extension design)}
\label{sec:e3e6}

We provide complete experiment protocols to close remaining external-validity gaps:
\begin{itemize}
  \item \textbf{E3 AP2 case study} (agent payments): compare vanilla AP2, AP2 + prompt/DLP shields, and AP2 + SecureClaw (with/without SAP), measuring unauthorized mandate rate, sensitive-data leakage, utility, and latency \citep{ap2spec2025,whisperswealth2026}.
  \item \textbf{E4 AgentDojo exfiltration extension}: reproduce data-flow exfiltration tasks and compare vanilla, shielded, and SecureClaw variants under the same task manifest \citep{alizadeh2025simple,agentdojo2024}.
  \item \textbf{E5 VPI-Bench subset}: stress-test computer-use scenarios where effects can be mediated by executor; unmediated UI side effects are reported explicitly as scope limits \citep{vpibench2025}.

\end{itemize}

These protocols are specified with baseline mapping and fairness constraints (threat-model alignment, no label-oracle defenses), and are included to avoid over-claiming beyond measured evidence.

\section{Limitations and Discussion}
\label{sec:limits}

SecureClaw still depends on trusted gateway and executor.
SAP requires non-collusion between policy servers.
SCS is conditional on a valid capsule mediation contract.
If an agent can trigger unmediated OS/UI effects outside the executor path, NBE does not apply; such paths must be wrapped by a mediated executor to inherit the guarantee.
In addition, unkeyed request-hash observability can permit offline guessing on low-entropy fields; our claims therefore assume sufficient conditional min-entropy (or explicit residual-risk accounting) for bound request inputs.

Threat-model mismatch is a practical review risk: native guardrails are meaningful in Threat Model A but are not designed to satisfy Threat Model B's non-bypassable requirements.
We therefore separate these evaluations explicitly.

\section{Related Work and Positioning}
\label{sec:related}

Prompt-injection attacks and tool-chain leakage in LLM agents are well established \citep{greshake2023indirect,injecagent2024,toolselect2025,stac2025,agentdojo2024,agentleak2026}.
Runtime defenses such as DRIFT, IPIGuard, and AgentArmor improve mediated behavior but do not by themselves instantiate an executor-side non-bypassable commit boundary \citep{drift2025,ipiguard2025,agentarmor2025}.

\paragraph{Positioning versus Faramesh.}
Faramesh's AAB and SecureClaw share the execution-boundary philosophy \citep{faramesh2026}.
Our claim is orthogonal in two dimensions:
(1) \emph{control-plane privacy}: SAP formalizes and proves single-auditor transcript privacy for outsourced policy checks;
(2) \emph{confidentiality under compromised runtime}: handleized plaintext confinement and delegation-aware hash-context binding are integrated into the same commit-security line.
Hence the key difference is not ``decision-before-execution'' alone, but also ``who can learn decision inputs'' and ``how cross-principal reuse is prevented.''

\section{Conclusion}
\label{sec:conclusion}

SecureClaw is centered on one proposition: robust agentic execution security requires a non-bypassable commit boundary, and practical outsourced policy enforcement requires a leakage-contracted privacy model.
Our measured results support both parts on official benchmark coverage and single-auditor distinguishability tests.
The provided E3--E6 protocols extend this line toward AP2, AgentDojo exfiltration workflows, computer-use stress settings, and enterprise control-plane privacy-sink baselines while preserving threat-model fairness.
